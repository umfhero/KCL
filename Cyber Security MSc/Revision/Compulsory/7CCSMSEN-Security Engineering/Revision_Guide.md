# Security Engineering (7CCSMSEN) Revision Guide

**Credits:** 15 | **Semester:** Autumn | **Assessment:** 70% Exam / 30% Coursework

## Module Overview
This is the most technically hands-on Autumn module. They will cover:
- Command injection, x86 assembly, stack-based buffer overflows, shellcode writing, format string vulnerabilities
- Memory corruption mitigations
- Secure software development (threat modelling, defence-in-depth)
- Static analysis and symbolic execution

## Preparation & Prerequisites
- **Prerequisites:** Prior experience with C/C++ and basic OS concepts is **essential**. Students without low-level programming experience traditionally struggle with the exploitation content.
- **Coursework:** Likely involves practical exploit development or secure code analysis.

## Reading List
- **Primary Text (FREE):** Ross Anderson's *Security Engineering: A Guide to Building Dependable Distributed Systems* (3rd edition, 2020). Available entirely for free as chapter PDFs from cl.cam.ac.uk/archive/rja14/book.html.

They should use this folder to store practice exploits, assembly notes, C/C++ revision materials, and chapter summaries from Anderson's book.
